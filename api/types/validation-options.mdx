## ValidationOptions Interface

Options to customize the validation behavior for single email validation.

```typescript
export interface ValidationOptions {
  /** Enable deep SMTP verification (default: false) */
  deep?: boolean;
  /** Check MX records (default: true) */
  checkMx?: boolean;
  /** Use DeBounce API for disposable detection (default: true) */
  useDeBounce?: boolean;
  /** Timeout for DNS lookups in seconds (default: 5) */
  timeout?: number;
}
```

## Fields

<ResponseField name="deep" type="boolean" default={false}>
  Enable deep SMTP verification to check if the mailbox actually exists. This makes an API call to Sniffmail and requires an API key.
  
  - `false` (default): Quick validation with syntax, disposable, and MX checks only
  - `true`: Includes full SMTP mailbox verification via Sniffmail API
</ResponseField>

<ResponseField name="checkMx" type="boolean" default={true}>
  Whether to verify that the domain has valid MX (mail exchange) records configured.
  
  - `true` (default): Check for MX records
  - `false`: Skip MX record validation
</ResponseField>

<ResponseField name="useDeBounce" type="boolean" default={true}>
  Whether to use the DeBounce API for additional disposable email detection.
  
  - `true` (default): Query DeBounce API for disposable domains
  - `false`: Skip DeBounce API check (faster but may miss some disposables)
</ResponseField>

<ResponseField name="timeout" type="number" default={5}>
  Timeout for DNS lookups in seconds. Increase if you experience timeout errors on slow networks.
  
  Valid range: 1-30 seconds
</ResponseField>

## Examples

### Default (Quick Mode)

```typescript
import { validateEmail } from '@sniffmail/sdk';

// Uses default options: quick validation only
const result = await validateEmail('user@example.com');
```

### Deep SMTP Verification

```typescript
// Enable deep mode for full mailbox verification
const result = await validateEmail('user@example.com', {
  deep: true
});

if (result.smtp) {
  console.log('SMTP deliverable:', result.smtp.is_deliverable);
}
```

### Skip MX Check

```typescript
// Validate syntax and disposable status only
const result = await validateEmail('user@example.com', {
  checkMx: false
});
```

### Custom Timeout

```typescript
// Increase timeout for slow networks
const result = await validateEmail('user@example.com', {
  timeout: 10
});
```

### Combined Options

```typescript
// Deep validation with custom settings
const result = await validateEmail('user@example.com', {
  deep: true,
  checkMx: true,
  useDeBounce: true,
  timeout: 8
});
```

## BatchValidationOptions Interface

Extends `ValidationOptions` with additional options for batch validation.

```typescript
export interface BatchValidationOptions extends ValidationOptions {
  /** Max concurrent API calls for deep mode (default: 5) */
  concurrency?: number;
}
```

## Additional Fields

<ResponseField name="concurrency" type="number" default={5}>
  Maximum number of concurrent API calls when using deep mode with batch validation.
  
  - Lower values (1-3): Slower but reduces API load
  - Default (5): Balanced performance
  - Higher values (10+): Faster but may hit rate limits
  
  Only applies when `deep: true` is enabled.
</ResponseField>

### Inherits All ValidationOptions

<Expandable title="Inherited fields from ValidationOptions">
  <ResponseField name="deep" type="boolean" default={false}>
    Enable deep SMTP verification for all emails in the batch
  </ResponseField>
  
  <ResponseField name="checkMx" type="boolean" default={true}>
    Check MX records for all emails
  </ResponseField>
  
  <ResponseField name="useDeBounce" type="boolean" default={true}>
    Use DeBounce API for all emails
  </ResponseField>
  
  <ResponseField name="timeout" type="number" default={5}>
    DNS lookup timeout in seconds
  </ResponseField>
</Expandable>

## Batch Examples

### Default Batch Validation

```typescript
import { validateBatch } from '@sniffmail/sdk';

const emails = [
  'user1@example.com',
  'user2@temp-mail.org',
  'user3@gmail.com'
];

const result = await validateBatch(emails);
console.log(`Valid: ${result.summary.valid}/${result.summary.total}`);
```

### Deep Batch with Custom Concurrency

```typescript
// Process 10 emails at a time
const result = await validateBatch(emails, {
  deep: true,
  concurrency: 10
});
```

### Conservative Rate Limiting

```typescript
// Process 2 emails at a time to avoid rate limits
const result = await validateBatch(emails, {
  deep: true,
  concurrency: 2,
  timeout: 10
});
```

### Fast Batch (No Deep Checks)

```typescript
// Quick syntax and disposable checks only
const result = await validateBatch(emails, {
  deep: false,
  useDeBounce: true
});
```

## Performance Guidelines

### Quick Mode (deep: false)
- **Speed**: ~10-50ms per email
- **Cost**: Free
- **Checks**: Syntax, disposable domains, MX records
- **Best for**: Real-time form validation, large batches

### Deep Mode (deep: true)
- **Speed**: ~1-3 seconds per email
- **Cost**: Uses API credits
- **Checks**: Everything + SMTP mailbox verification
- **Best for**: Sign-up verification, email list cleaning

### Concurrency Recommendations

| Batch Size | Recommended Concurrency |
|-----------|------------------------|
| 1-10      | 5 (default)            |
| 10-100    | 5-10                   |
| 100-1000  | 10-15                  |
| 1000+     | 15-20                  |
